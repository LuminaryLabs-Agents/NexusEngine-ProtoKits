# Experiment Map

This file tracks the canonical experiment portfolio and the domains each route validates.

## Standing constraints

- Experiments should harden toward about 20 canonical routes.
- The number 20 is a guiding target, not a brittle hard constraint.
- Prefer merging overlapping experiments, features, and routes into stronger cumulative domain pressure.
- Experiments should validate reusable ProtoKits, not own reusable kit implementation.
- Route work should keep hosts thin: presets, bridges, manifests, docs, tests, and renderer-only presentation.
