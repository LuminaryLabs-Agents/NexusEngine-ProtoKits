## Intent alignment

The new VR platformer suite is directionally aligned only if it remains a composed set of scoped domains and descriptor/adaptor boundaries rather than a monolithic game kit. Its README says the host owns Canvas/WebGL/Three drawing, actual WebXR/OpenXR session, raw runtime handles, and frame presentation; the kits own platformer state, collision meaning, camera/render descriptors, XR pose/input descriptors, spatial board transforms, and comfort policy.

## DSK boundary clarity

Boundary clarity improves if this family is treated as a higher-level `spatial-platformer-loop` above these child boundaries:

- platformer level/avatar/physics/collision/object/objective sequence
- platformer camera/render/effects/parallax descriptors
- XR pose/input descriptors
- spatial anchor/game board transforms
- XR comfort and render-plan descriptors
- stereoscopic render descriptors composed beside the suite

Do not promote the maximum-feature suite as a Core-ready DSK. Promote or incubate the child domains only after they expose clear resources/events/methods/snapshots/descriptors and deterministic replay.
